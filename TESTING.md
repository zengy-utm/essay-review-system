## 🎯 5分钟快速验证

### 前置准备
```bash

cd /Users/yea/Desktop/"cloud code"

# 2. 查看项目结构
ls -la
# 应该看到: my-website/, backend/, README.md, start.sh
```

### 启动服务

#### 方式1: 使用快速启动脚本（推荐）
```bash
chmod +x start.sh
./start.sh
```

#### 方式2: 手动启动（两个终端窗口）
```bash

cd backend
npm install
cp .env.example .env
# 编辑 .env，填入 GEMINI_API_KEY
npm start

# 终端2 - 前端
cd my-website
npm install
npm run dev
```

## 🔧 常见测试问题

### 问题1: 后端无法连接
```
错误信息: Failed to connect to http://localhost:3001
```
**解决：**
1. 确保后端进程运行: `lsof -i :3001`
2. 检查端口是否被占用
3. 查看后端控制台是否有错误

### 问题2: API Key错误
```
error: 401 Unauthorized
```
**解决：**
1. 检查 backend/.env 中的 GEMINI_API_KEY
2. 验证API Key有效性
3. 或使用模拟数据测试（见下面）

### 问题3: 文件上传失败
**检查：**
1. 文件格式是否支持
2. 文件大小是否超过限制
3. 浏览器控制台查看错误信息(F12)

### 问题4: 页面样式混乱
**解决：**
1. 清除浏览器缓存 (Cmd+Shift+Delete)
2. 硬刷新页面 (Cmd+Shift+R)
3. 检查CSS文件是否正确加载 (DevTools > Network)

## 📈 性能测试

### 前端加载速度
1. 打开 DevTools (F12)
2. Performance 标签
3. 记录初始加载时间（应 < 2秒）

### 后端响应时间
1. 在 DevTools Network 标签监控
2. 提交评审并记录响应时间
3. 使用Gemini: 30-60秒
4. 使用模拟数据: < 1秒
